# 订单接口性能测试报告

## 一、测试场景

* 并发线程数：10
* 总请求数：约 500
* 接口地址：POST https://postman-echo.com/post
* 测试数据：{"order\_id": "JM\_TEST\_001", "amount": 500, "status": "CREATED"}

|指标|数值|
|-|-|
|平均响应时间|337 ms|
|中位数响应时间|320 ms|

|最小响应时间|237 ms|
|最大响应时间|1968 ms|
|吞吐量 (TPS)|\~21/sec|
|错误率|0.00%|

## 三、结果分析

1. **稳定性**：错误率为 0%，接口在 10 并发下完全稳定，无请求失败。
2. **响应性能**：平均响应时间 337 ms，90% 请求在 371 ms 内完成，符合一般业务接口性能要求。
3. **吞吐量**：TPS 约 21，代表接口可稳定支撑每秒约 21 次请求，具备良好的抗压能力。
4. **波动分析**：最大响应时间 1968 ms，99% 线为 1061 ms，存在偶发高延迟，推测为公共测试服务限流导致。

## 四、结论

* 接口在 10 并发场景下表现优异，功能稳定、性能达标。
* 偶发高延迟为测试环境特性，非接口本身问题，生产环境可进一步优化。
